# Seguranca

## Como reportar vulnerabilidades

Se encontrar uma falha de seguranca, envie um e-mail para:

```text
rodrigoalgeri.ms@gmail.com
```

Inclua:

- Descricao do problema.
- Passos para reproduzir.
- Impacto esperado.
- Versao afetada.

Evite divulgar publicamente a falha antes de haver tempo para analise e correcao.

## Principios do projeto

- Restaurar apenas URLs consideradas seguras.
- Manter dados no armazenamento local do navegador.
- Declarar somente permissoes necessarias no `manifest.json`.
